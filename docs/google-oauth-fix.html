תיקון שגיאת Google OAuth

שגיאה: redirect_uri_mismatch (400)

מה קרה?

Google לא מכיר את ה-Supabase callback URL כ-redirect URI מורשה. צריך להוסיף אותו ב-Google Cloud Console.

שלבים לתיקון

1
פתח את Google Cloud Console:
console.cloud.google.com/apis/credentials
2
בחר את הפרויקט שבו הגדרת את OAuth Client (כנראה "kidushishi" או "bayit-beseder")
3
לחץ על OAuth 2.0 Client ID הקיים (מסוג "Web application")
4
בסעיף "Authorized redirect URIs" — ודא שה-URIs הבאים קיימים:

https://uqumzjmyejlhoyliyesu.supabase.co/auth/v1/callback

וגם (לפיתוח מקומי):
http://localhost:3000/callback
5
בסעיף "Authorized JavaScript origins" — ודא:

https://www.bayitbeseder.com

https://uqumzjmyejlhoyliyesu.supabase.co
6
לחץ Save — זה עשוי לקחת כמה דקות להתעדכן

חשוב: בדוק גם ב-Supabase Dashboard

ב-Supabase Dashboard → Auth → Providers → Google:

  • ודא ש-Client ID ו-Client Secret מעודכנים
  • ודא ש-"Site URL" הוא https://www.bayitbeseder.com
  • ודא ש-"Redirect URLs" כולל https://www.bayitbeseder.com/callback

למה זה קורה?

Supabase OAuth flow:

1. האפליקציה שלך → Google (login)

2. Google → Supabase callback (uqumzjmyejlhoyliyesu.supabase.co/auth/v1/callback)

3. Supabase → האפליקציה שלך (bayitbeseder.com/callback)


Google צריך לדעת על שלב 2 — ה-Supabase URL. בלי זה → redirect_uri_mismatch
